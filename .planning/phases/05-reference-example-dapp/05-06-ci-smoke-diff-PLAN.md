<objective>
Build a CI smoke-diff job that proves the committed `examples/confidential-token/` did not silently drift from current skill output. On every PR, re-run the skills with the same inputs (recorded in `.gsd-snapshot.json`), diff the resulting key files against the committed example, and fail CI if drift exceeds the allowlist.

Purpose: EXAMPLE-05 — without this, a maintainer can quietly hand-edit the example and the dogfooding claim becomes a lie. The smoke-diff is the enforcement.
Output: A green CI badge on `main`, a runnable local diff command, and a clear allowlist policy.
</objective>

Allowlist patterns (`scripts/example-smoke-diff.allowlist.json`):
```json
{
  "patterns": [
    { "file": "packages/contracts/hardhat.config.ts", "regex": "^// Generated by /zama-init at .+$", "reason": "timestamp comment" },
    { "file": "packages/contracts/package.json", "regex": "\"version\":\\s*\"[^\"]+\"", "reason": "version field auto-bumps" },
    { "file": "packages/contracts/contracts/Token.sol", "regex": "^// HCU budget: .+$", "reason": "version-tagged HCU comment" }
  ],
  "comment": "Drift outside these patterns fails CI. Adding patterns requires reviewer sign-off in the PR."
}
```

Driver flow (`scripts/example-smoke-diff.mjs`):
1. Read `examples/confidential-token/.gsd-snapshot.json` → extract `scaffold_inputs`.
2. mkdir `/tmp/zama-smoke-diff-<sha>` and `cd` there.
3. Invoke each skill's runtime entry point with the recorded inputs:
   - `node <repo>/plugins/zama-skills/skills/init/scripts/scaffold.ts --inputs '{"name":"confidential-token","monorepo":true,"pm":"pnpm"}'`
   - `node <repo>/plugins/zama-skills/skills/contract/scripts/generate.ts --inputs '{"name":"Token","base":"erc7984","schema":[],"decryptionPath":"user","metadata":{"name":"Confidential Demo Token","symbol":"cDEMO","decimals":6}}'`
   - `node <repo>/plugins/zama-skills/skills/frontend/scripts/generate.ts --inputs '{"contract":"packages/contracts/contracts/Token.sol","withWagmi":true}'`
4. For each key file in `<interfaces>`, run a normalized diff:
   - Strip lines matching allowlist patterns from BOTH sides.
   - For `package.json` files, parse JSON, extract just `dependencies` + `devDependencies`, sort keys, re-serialize.
   - Compare with `node:diff` (no extra deps) — produce a unified diff if mismatched.
5. Exit 0 if all diffs are empty post-normalization; exit 1 with the unified diff in stdout otherwise.

CI workflow (`.github/workflows/example-smoke-diff.yml`):
- Triggers: `pull_request`, `push: [main]`.
- Steps: checkout, setup-node@20, `pnpm install --frozen-lockfile` at repo root, `node scripts/example-smoke-diff.mjs`.
- Cache: pnpm store.
- Timeout: 10 min.
- On failure: print the diff in the job log + a friendly message: "If this drift is intentional (e.g., you bumped a skill template), add a pattern to scripts/example-smoke-diff.allowlist.json explaining why. Otherwise re-run /zama-init etc. and commit the fresh output."

Self-test: `scripts/example-smoke-diff.test.mjs` exercises the diff function on synthetic fixtures (matching/mismatching content, allowlist-stripped, package.json sort).
